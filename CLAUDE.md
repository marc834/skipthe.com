# Notes for AI assistants

Read this before making changes.

## What this project is

skipthe.com is a hyperlocal news aggregator. It runs as a periodic Python job on a Google Cloud VM, scrapes a broad set of hyperlocal sources, asks OpenAI whether each item is worth surfacing, and publishes a browsable website + RSS feeds that nginx serves.

The **website is the primary product**; RSS is a secondary surface for the small minority of readers who use feed readers. AI moderation is the editorial layer — there is no human review queue. A visible disclaimer says so.

It is **not** a CMS, not a user-generated platform (yet), and not a real-time service. The whole pipeline is a cron-driven batch.

## Where things live

- `main.py` — single entrypoint, runs the full pipeline
- `app/config/sources.yaml` — neighborhoods, keywords, feed sources. Most "add a thing" requests start here.
- `app/collectors/` — `rss_collector.py` (feedparser) and `webpage_collector.py` (requests + bs4 link harvest). New collector types go here.
- `app/ai/classifier.py` — OpenAI call, JSON-mode response. Prompt is the moderation contract.
- `app/db/database.py` — SQLite, single `items` table, dedup by URL.
- `app/rss/generate_feeds.py` — writes one XML per category into `data/feeds/<neighborhood>/`.
- `app/site/` — static site. Today: a single `index.html`. Soon: pipeline-generated HTML pages — see `docs/WEBSITE.md`.
- `data/` — SQLite DB and generated artifacts. Gitignored.
- `docs/` — long-form documentation.

## Conventions to keep

- **Single source of truth for sources/keywords**: `app/config/sources.yaml`. Don't sprinkle source URLs in code.
- **URL scheme is path-based**: `skipthe.com/nocatee/`, never `nocatee.skipthe.com`. Decided.
- **Conservative AI prompt**: the classifier defaults to *exclude*. If you tune the prompt, preserve "be conservative", "never present allegations as fact", and the JSON schema contract — `generate_feeds.py` reads `include`, `category`, `summary`, `credibility` directly.
- **Dedup is by URL** (`UNIQUE` constraint). Collectors can re-emit the same URL freely; `INSERT OR IGNORE` handles it.
- **Feeds and HTML are regenerated from scratch each run** — they're a view of the DB, not append-only files. Don't try to mutate in place.
- **Categories live in two places** (yes, it's a smell): `classifier.py` SYSTEM prompt and `generate_feeds.py` `CATEGORIES`. Keep them in sync if you add one.
- **AI disclaimer must be visible** on every page and in every RSS feed description. It's the editorial cover.

## Things to avoid

- Don't add a web framework (Flask/FastAPI) to serve dynamic content unless the user asks. Static + RSS is the product. The website is generated by the pipeline, not served by an app server.
- Don't introduce per-source bespoke parsers in `webpage_collector.py`. If a source needs custom logic, add a new collector module in `app/collectors/` and dispatch on the `type:` field in `sources.yaml`.
- Don't store secrets in `sources.yaml` or commit `.env`.
- Don't write the OpenAI key into logs.
- Don't expand the SQLite schema speculatively. New columns should be tied to a real consumer.
- Don't add user accounts, comments, or interactive features without an explicit ask. The contact form (take-down requests) is the only inbound surface today.
